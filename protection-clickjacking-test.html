Clickjacking Simulation: "Enter Raffle" Button

What this demonstrates:
This test simulates a clickjacking attack, where a legitimate-looking button ("Overlay button") is visually presented to the user, but clicking it actually interacts with a button or input behind it — in this case, inside an iframe of Okta’s login page.

The attacker uses styling and pointer-events: none to trick the user into thinking they're clicking the raffle button, while in reality it's submitting information or clicking a login action underneath.
